Using the Git Command on Bash Terminal

By:Andy Kukuc

The Advantages of using the Git Command.

Using the Git commands or seeing a terminal will be hard for a lot of people. In reality using the git commands and the bash terminal to commit and push changes is very easy to do. Also if you learn to do git the hard way then you become more skilled in programming and coding. You teach yourself to type faster and to make less mistakes since it is all case sensitive.

The Three Basic Commands to using the Git Command.

There are three basic commands when it comes to using Git.

  1. Git Add
  2. Git Commit -m "String"
  3. Git Push Origin Master

1. Git Add

The git add command is basically arming the file to get ready to be sent to github.com The only way it works is when you have a file ready. If you make directorys and try the git add command it will not work. If you make a directory say: (C:\Users\Andy\NewFile\test.txt). You must be in the directory where the test.txt file is. Then you type in git add test.txt click enter and it will save the file for upload.

2. Git Commit -m "String"

The git commit -m command sends a message with the file and the directory to github.com. It is important that when you're ready to make a commit or made any modifications its best to leave behind a comment that says what you did. So say in example, I modified a program to do one thing. I would input in my terminal git commit -m "Made a change in program to do math faster". Then you click enter and you will get a prompt that comes up but you can ignore all the stuff that comes up.

3. Git Push Origin Master

The git origin master is what sends all your files and all your commits to github.com. The command pushes the files like github desktop and it will ask for your git username and password. When you hit enter everything will be sent to your repository. The file will upload and will appear on your github repository.
